# 智能周报 Agent 需求文档

> 这是一个用于演示 Markdown 结构化编辑能力的测试文档。
> 重点测试：读取 section、替换 section、在标题后插入内容、列出/更新代码块、列出/更新任务。

## 1. 背景

团队每周需要从项目进展、风险、待办事项中整理周报。当前主要依赖人工汇总，容易遗漏信息，也难以保持格式一致。

### 目标

- 自动汇总项目进展
- 提取风险和阻塞项
- 生成统一格式的周报
- 支持人工局部修改

### 非目标

- 不做复杂排版
- 不接入 IM 实时消息
- 不生成绩效评价

## 2. 用户场景

### 场景一：生成周报

用户上传项目记录后，Agent 根据模板生成一份周报草稿。

### 场景二：更新风险

用户只想修改“风险”部分，而不是重写整份文档。

### 场景三：勾选任务

用户希望 Agent 能把某个任务从未完成改成已完成。

## 3. 功能需求

### 3.1 读取指定章节

Agent 应能根据标题路径读取指定 section。

示例请求：

```json
{
  "heading_path": ["3. 功能需求", "3.1 读取指定章节"]
}
```

### 3.2 替换指定章节

Agent 应能替换某个 section 的正文，并保留标题。

适合测试：

- 替换 `## 5. 风险`
- 替换 `### 3.2 替换指定章节`
- 替换 `## 7. FAQ`

### 3.3 插入新章节

Agent 应能在指定 heading 后插入新内容。

示例：

```markdown
## 4. 数据来源

新增的数据来源说明。
```

### 3.4 更新代码块

Agent 应能列出代码块，并更新指定代码块内容。

```python
def generate_weekly_report(project_name: str) -> str:
    return f"{project_name} weekly report"
```

## 4. 数据来源

| 来源 | 说明 | 是否必需 |
|---|---|---|
| 项目记录 | 每日进展和变更 | 是 |
| 任务列表 | 当前待办和完成情况 | 是 |
| 风险登记表 | 风险、阻塞和负责人 | 否 |

## 5. 风险

### 目标

这里的“目标”标题用于简单测试重名标题，不需要复杂处理。

- 风险一：输入信息不完整，导致周报遗漏。
- 风险二：自动总结过度简化，丢失关键上下文。
- 风险三：任务状态与真实项目状态不同步。

## 7. FAQ

### Q1：为什么要按 section 编辑？

因为这样可以减少误改无关内容的风险。

### Q2：标题重名怎么办？

演示版本中可以简单要求使用完整标题路径。
