# 智能周报 Agent 测试文档

> 本文档用于测试 Markdown Agent 的基础结构化编辑能力。
> 重点测试：读取 section、替换 section、在指定标题后插入内容、更新任务列表、基础格式化。

## 1. 背景

团队每周需要整理项目周报，内容包括项目进展、风险、待办事项和下周计划。当前主要依赖人工汇总，容易遗漏信息，也难以保持格式一致。

### 1.1 目标

* 自动汇总项目进展。
* 提取风险和阻塞项。
* 生成统一格式的周报。
* 支持人工局部修改。

### 1.2 非目标

* 不生成绩效评价。
* 不做复杂排版。
* 不接入即时通信工具。

## 2. 用户场景

### 场景一：生成周报

用户上传项目记录后，Agent 根据模板生成一份周报草稿。

### 场景二：更新风险

用户只想修改“风险”部分，而不是重写整份文档。

### 场景三：插入补充说明

用户希望 Agent 能在指定章节后插入新的说明内容。

## 3. 功能需求

### 3.1 读取指定章节

Agent 应能根据标题路径读取指定 section。

示例请求：

```json
{
  "heading_path": ["3. 功能需求", "3.1 读取指定章节"]
}
```

### 3.2 替换指定章节

Agent 应能替换某个 section 的正文，并保留该 section 的标题。

适合测试：

* 替换 `## 5. 风险`
* 替换 `### 3.2 替换指定章节`

### 3.3 插入新章节

Agent 应能在指定 heading 后插入新内容。

示例：

```markdown
### 新增说明

这里是新增说明内容。
```

## 4. 数据来源

周报 Agent 主要读取以下数据来源：

| 来源    | 说明        | 是否必需 |
| ----- | --------- | ---- |
| 项目记录  | 每日进展和变更   | 是    |
| 任务列表  | 当前待办和完成情况 | 是    |
| 风险登记表 | 风险、阻塞和负责人 | 否    |

## 5. 风险

* 输入信息不完整，导致周报遗漏。
* 自动总结过度简化，丢失关键上下文。
* 任务状态与真实项目状态不同步。

## 7. FAQ

### Q1：为什么要按 section 编辑？

因为这样可以减少误改无关内容的风险。

### Q2：标题重名怎么办？

演示版本中可以简单要求使用完整标题路径。
